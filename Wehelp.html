描述一件產生明顯負面情緒的經歷，如何處理？

Emotion & Coping

剛轉職成為工程師時，面對工作上許多在訓練班沒有學到的新技術與流程，常常感到挫折與不安。每當遇到問題，我必須靠自我學習、查閱文件與資源，並反覆測試程式碼以確保上線順利。 曾有一次因測試遺漏某種情境，導致程式上線後出現異常。當下我先冷靜分析，並依循流程找到問題根因，提出短期替代方案，確保系統正常運行，接著修復程式並重新完整測試後再上線。

  • 找到問題根因：立即確認錯誤發生原因，避免盲目操作。
  • 替代解決方案：設計短期措施，讓系統維持穩定運作。
  • 修復程式並謹慎測試：完成程式修復後進行完整 End-to-End 測試。
  • 記取教訓：將每次錯誤詳細紀錄，建立團隊共享守則，避免重複發生。

透過流程化處理，我將情緒轉化為行動，也在回顧會議中提出可被團隊沿用的守則。這段經歷讓我更重視 數據品質、監控與自動化測試，並促使我進修 ML 與 GA4 課程，持續提升專業能力與問題解決的自信心。

最想使用自己開發的深度學習模型解決什麼問題？

DL Problem

作為未來的數據分析師，我希望透過深度學習模型整合不同來源的金融數據，提供更精準且個人化的投資建議，並將分析結果轉化為可操作的策略。

我曾經修習台大「Python 財務金融資料分析與處理」課程，累積了以下能力：

  • 金融資料抓取與清洗：利用 API 與爬蟲整合市場數據，進行資料清理與格式轉換。
  • 資料分析與視覺化：用 Pandas、NumPy、Matplotlib、Seaborn 處理與呈現金融數據，分析趨勢與波動。
  • 投資組合與風險管理：計算報酬率、波動率，模擬投資組合績效與回測。
  • 機器學習在金融應用：使用 scikit-learn 建立監督式與非監督式模型進行預測與分群分析。

基於這些基礎，我希望使用 深度學習模型（如 LSTM / RNN） 處理時間序列金融資料，結合 聯邦學習（Federated Learning），讓不同資料來源在保護隱私的前提下，共同訓練模型，達到以下目標：

  • 提升模型泛化能力：多元資料融合，讓策略在不同市場環境下更穩健。
  • 降低風險與偏差：減少單一資料來源帶來的偏差。
  • 知識共享與策略創新：各機構或資料源的專業知識能整合到模型中，促進創新。
  • 個人化投資規劃：模型可根據投資人的風險承受度與目標，提供可行的投資組合建議。

透過這套模型，我希望能將 金融資料分析能力與深度學習技術 結合，實際落地於投資規劃與策略優化中，協助投資人或機構做出更理性且有效的決策，朝數據分析師方向持續邁進。

若無法達到 OpenAI 程度，為何要學習基礎模型的實作？

Why Foundation Models
  • 理解原理才能應用：雖然無法達到 OpenAI 的規模，但理解基礎模型的運作原理，能幫助我正確使用現有的 API 與工具。
  • 可定制化應用：基礎模型熟悉後，可依照特定業務或金融資料需求進行微調（Fine-tuning），打造專屬模型。
  • 建立技術底層能力：掌握資料處理、模型訓練、損失函數、梯度下降等核心技術，有助於解決實務問題。
